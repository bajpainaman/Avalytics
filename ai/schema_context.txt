## Avalytics Database Schema

Table: transactions
- tx_hash (TEXT, PRIMARY KEY)
- block_number (INTEGER)
- from_address (TEXT) - sender wallet
- to_address (TEXT) - receiver wallet
- value (INTEGER) - amount in wei
- gas_price (INTEGER)
- gas_used (INTEGER)
- status (INTEGER) - 1 for success, 0 for failure
- timestamp (DATETIME)
- input_data (TEXT)

Table: wallet_profiles
- wallet_address (TEXT, PRIMARY KEY)
- first_seen (DATETIME)
- last_active (DATETIME)
- total_txs (INTEGER) - total transaction count
- total_volume_wei (INTEGER) - total value sent
- unique_contracts (INTEGER) - number of unique contracts interacted with
- is_whale (INTEGER) - 1 if whale, 0 otherwise
- is_bot (INTEGER) - 1 if bot, 0 otherwise
- is_dex_user (INTEGER) - 1 if DEX user, 0 otherwise
- is_nft_collector (INTEGER) - 1 if NFT collector, 0 otherwise
- avg_tx_value_wei (INTEGER)
- behavior_score (REAL)
- last_updated (DATETIME)

Table: wallet_tags
- id (INTEGER, PRIMARY KEY)
- wallet_address (TEXT)
- tag (TEXT) - cohort tag like 'whale', 'dex_user', etc.
- confidence (REAL)
- created_at (DATETIME)

Table: event_logs
- id (INTEGER, PRIMARY KEY)
- tx_hash (TEXT)
- block_number (INTEGER)
- log_index (INTEGER)
- address (TEXT) - contract address
- topic0 (TEXT) - event signature
- topic1, topic2, topic3 (TEXT) - indexed parameters
- data (TEXT) - event data
- timestamp (DATETIME)
